# VAST for Operations Leaders

## What this means for Operations

Operations already thinks systemically — processes, cadences, scaling infrastructure. VAST gives this thinking a name and a place in the organizational sequence.

## What changes

**Your work IS Architecture.** Org design, operating rhythms, cross-functional alignment, scaling infrastructure — this is the Architecture layer. VAST makes it explicit and gives it sequencing priority: Architecture before Strategy, not after.

**OKR decomposition becomes sharper.** VAST splits OKRs into three levels: Outcomes (what domains matter), Use Cases (where to invest), Outputs (what to produce). For Operations, this means: "operational efficiency" is an Outcome. "Automate procurement workflow" is a Use Case. "Deploy procurement bot by March" is an Output. Naming the levels prevents OKRs from collapsing into task lists.

**Values become your enforcement tool.** "Execution > Frameworks" is itself a value — and VAST says values are architectural constraints. Operations can use this: values aren't posters, they're governance criteria.

## What doesn't change

- Operations still owns process excellence
- Existing cadences (QBRs, planning cycles) continue
- You still facilitate cross-org alignment
