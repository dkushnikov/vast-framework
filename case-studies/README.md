# Case studies

Concrete applications of VAST to real problems. Each case study has the same shape:

1. **Context** — what was the problem, what was the current state before VAST.
2. **Layers** — how the four layers mapped to the specific problem (what was Vision, Architecture, Strategy, Tactics, who owned each).
3. **Challenge flow in action** — examples of signals entering through different layers and being processed.
4. **What VAST prevented** — specific dysfunctions that would have occurred without the sequencing.
5. **Artifacts** — pointers to real decisions, docs, session logs.
6. **Lessons** — what the application revealed about the framework itself.

## Adding a case study

When VAST is applied to a new problem domain, add a new file `NNN-short-name.md` following the structure above. Number sequentially. Keep each case under ~500 lines; split to sub-files if the application warrants.
