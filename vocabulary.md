# VAST Vocabulary — Multi-Level Reference

Definitions and links, without details. Learn the vocabulary, then go deep.

## Philosophical

| Term | One-line | Link |
|---|---|---|
| **VAST** | Sequencing philosophy: Vision → Architecture → Strategy → Tactics | [vast.md](vast.md) |
| **KosmOS** | VAST applied to life. "Happiness = properly tuned world model" | [github.com/dkushnikov/kosmos](https://github.com/dkushnikov/kosmos) |
| **Sequencing principle** | Commitment priority: fix Vision before Architecture before Strategy before Tactics | [vast.md → "The philosophy"](vast.md) |
| **Eudaimonia** | Aristotelian flourishing through reason and virtue, not hedonic pleasure | [Wikipedia](https://en.wikipedia.org/wiki/Eudaimonia) |
| **Bidirectionality** | V→A→S→T = logical priority, not temporal lockstep. Arrows go both ways | [vast.md](vast.md) |
| **Complexity threshold** | VAST adds value above a certain complexity. Simple systems don't need it | [standard-framework.md](standard-framework.md) |
| **Extension, not replacement** | VAST extends standard frameworks, doesn't replace them | [standard-framework.md](standard-framework.md) |
| **Codification, not invention** | VAST names existing practice (like Agile Manifesto named iterative dev) | [standard-framework.md → "Prior art"](standard-framework.md) |

## Prior art (quick reference)

| Framework | What it contributes to VAST | Link |
|---|---|---|
| **Balanced Scorecard** | 4 perspectives cascade. Closest Architecture + Strategy + Tactics | [Wikipedia](https://en.wikipedia.org/wiki/Balanced_scorecard) |
| **Wardley Maps** | Value chain visibility + evolution axis. Best Architecture visualization | [Wikipedia](https://en.wikipedia.org/wiki/Wardley_map) |

| **PERMA** | 5 pillars of wellbeing (Seligman). Empirically validated | [Wikipedia](https://en.wikipedia.org/wiki/PERMA_model) |
